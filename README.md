# Vagrant + Ansible example

> Do NOT use in production.

## How to run?

On local Windows, Linux, MacOS run Vagrant.

```bash
vagrant up
```

Connect to the Management server

```bash
vagrant ssh mgmt
```

## Create test users and keys

In `/vagrant/scripts` are scripts for data generation and Ansible Playbook.

Generate test users on `mgmt` server.

```bash
# mgmt server

# sudo /vagrant/scripts/generator/run.sh [usernames ...]

# For example:
sudo bash /vagrant/scripts/generator/run.sh alice bob charlie alex steve
```

This script generates ssh keys and `ansible/vars/ssh_keys.yml` file.
Private keys are in `~/keys`.

---

## Add locations for users

Let's generate locations for the created users.

```bash
# sudo bash /vagrant/scripts/generator/locations.sh [locations ...]

# For example:
sudo bash /vagrant/scripts/generator/locations.sh dns web

# Write the usernames
> dns: alex charlie alice
> web: alex 
```

## Run Ansible Playbook

Script uses absolute hard-coded paths, so runable from any path.

```bash
bash /vagrant/scripts/run_playbook.sh
```

## Test private keys from Management server

Let's try to connect to the `dns` host from the management server.

Our test keys generated by `root`, so we need create `ssh tunel` as `root`

```bash
sudo ssh alice@10.0.0.30 -i /home/vagrant/keys/alice
> alice@ns1:~$

sudo ssh charlie@10.0.0.30 -i /home/vagrant/keys/charlie
> charlie@10.0.0.30: Permission denied (publickey).
```
